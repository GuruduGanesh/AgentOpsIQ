# Inspiration Map

AgentOpsIQ is an original product direction inspired by public market patterns from the AI infrastructure and observability ecosystem. Do not copy private code, proprietary UI, private docs, exact workflows, branding, or confidential implementation details from any company.

## Product Themes

## Ryvn-Inspired Theme: Customer-Cloud Deployment

What to learn:

- Enterprises need customer-owned cloud and private deployment patterns.

AgentOpsIQ feature direction:

- AKS kit
- Terraform reference
- Helm chart
- BYOC deployment notes

## Superlog-Inspired Theme: OpenTelemetry-First Observability

What to learn:

- OTel can be the standard evidence layer for incidents.

AgentOpsIQ feature direction:

- OTel collector pack
- Trace correlation
- Deployment metadata
- Incident report generation

## Sentrial-Inspired Theme: Agent Reliability

What to learn:

- AI agents have unique failure modes.

AgentOpsIQ feature direction:

- Tool-call failures
- Retry-loop detection
- Agent SLOs
- Cost and latency controls

## Sazabi-Inspired Theme: AI-Native Incident Understanding

What to learn:

- SREs need actionable incident explanations, not more dashboards.

AgentOpsIQ feature direction:

- Incident timeline
- Root-cause hypothesis
- Local AI incident summary
- Remediation recommendations

## Metoro-Inspired Theme: Kubernetes-Native Context

What to learn:

- Runtime context is essential for reliable AI services.

AgentOpsIQ feature direction:

- Kubernetes event ingestion
- Resource checks
- Pod restart correlation
- AKS readiness checks
